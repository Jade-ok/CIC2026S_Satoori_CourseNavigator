# UBC Course Navigator

**Team Satoori** — UBC CIC Spring 2026 GenAI Hackathon

A personalized course planning tool for UBC Computer Science students, built on AWS. Students describe their career goal in natural language and receive a visual prerequisite roadmap with AI-generated course recommendations.

---

## Background

This project was built at the **UBC Cloud Innovation Centre (CIC) Spring 2026 GenAI Hackathon**, a one-day competition focused on building generative AI applications using AWS services. For our team, this was our first time building a cloud-native application on AWS — from IAM roles and Lambda functions to connecting a frontend to Bedrock.

Team members came in with frontend and backend experience but no prior AWS exposure. The hackathon was our introduction to thinking about cloud infrastructure, IAM permissions, and how to wire up managed AI services rather than running models locally.

## What it does

A student types their goal — e.g. *"I want to become an ML engineer"* — and optionally uploads a PDF transcript. The app returns:

- A **filtered course graph** showing only the courses on their goal path, organized by level (100–400)
- Visual states per course: completed, available now, AI-recommended, or locked behind prerequisites
- A **detail sidebar** per course with student reviews and related career events

---

## AWS Services Used

| Service | How it's used |
|---|---|
| **Amazon Bedrock** | Hosts Claude Sonnet 4.6 (`us.anthropic.claude-sonnet-4-6`). Invoked by Lambda to generate course recommendations as structured JSON. |
| **Amazon Bedrock Knowledge Base** | Indexes UBC course syllabi. The FastAPI backend retrieves relevant syllabus snippets and injects them into the prompt as context when Lambda is unavailable. |
| **AWS Lambda** | Runs the core recommendation logic — prompt construction, career track hint injection, Bedrock invocation, and JSON validation. |
| **Amazon S3** | Stores `courses.json` and `career_tracks.json`. Lambda fetches these at runtime so data can be updated without redeployment. |
| **AWS API Gateway** | Exposes the Lambda function as an HTTPS endpoint consumed by the FastAPI backend. |
| **AWS IAM** | Role-based access for Lambda to read S3 and invoke Bedrock. Configuring least-privilege policies was one of the main learning points. |

## Architecture

**Amazon Bedrock (Claude Sonnet 4.6)** is the AI engine at the center of the app. All course recommendations are generated by Claude via Bedrock.

The backend wires together three AWS components around it:

- **AWS Lambda** — serverless function that constructs the prompt and invokes Bedrock. Triggered via API Gateway from the FastAPI backend.
- **Amazon Bedrock Knowledge Base** — indexes UBC course syllabi. Retrieves relevant content at query time and injects it into the prompt as context before Claude is called.
- **Amazon S3** — stores `courses.json` and `career_tracks.json`. Both Lambda and the FastAPI backend read from S3 at runtime.

```
React (Vite + TypeScript)
    └── FastAPI
            ├── API Gateway → Lambda ──────────────────┐
            ├── Bedrock Knowledge Base (syllabus RAG) ─┼──→ Amazon Bedrock
            └── S3 (course data) ──────────────────────┘       (Claude Sonnet 4.6)
```

## What we built vs. what we didn't finish

**Working:**
- Natural language goal input with career track detection
- PDF transcript upload and course extraction via regex
- AI-generated course map filtered to the student's goal path
- Korean language input support (Claude responds in kind)
- Prerequisite chain inference — completed courses unlock downstream nodes

**Not fully implemented:**
- **Reddit student reviews** — the detail sidebar shows static mock data. The intent was to pull and summarize real posts from r/UBC using the Reddit API, but we ran out of time.
- **Career events** — similarly, the event cards are hardcoded. A real version would query a live events API or scraper.

## What we learned

- How to set up an end-to-end AWS workflow from scratch: S3 buckets, IAM roles, Lambda functions, and API Gateway — without prior cloud experience
- How IAM permission errors actually surface (and how to debug them)
- Prompt engineering for structured JSON output — getting a language model to reliably return a specific schema with no markdown fences
- How to use boto3 to manage AWS resources programmatically when the CLI isn't available
- The value of a fallback chain when cloud services are unreliable under hackathon time pressure

## Future development

- **Live Reddit integration** — summarize r/UBC and r/UBCcsss threads per course using the Reddit API + Bedrock summarization
- **Real events feed** — pull from UBC CS club calendars or Eventbrite
- **OR-logic prerequisites** — the current data model treats all prereqs as AND conditions; UBC courses often allow alternatives (e.g. MATH 152 *or* MATH 221)
- **More career tracks** — systems, HCI, and security tracks are defined in `career_tracks.json` but not yet tuned
- **Bedrock Knowledge Base** — index actual UBC course syllabi so recommendations can reference real course content, not just titles and prereq codes

## Local setup

**Backend**
```bash
cd backend
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # add AWS credentials
uvicorn main:app --reload
```

**Frontend**
```bash
cd frontend
npm install
npm run dev
```

Set `VITE_API_URL=http://localhost:8000` in `frontend/.env.local`.
